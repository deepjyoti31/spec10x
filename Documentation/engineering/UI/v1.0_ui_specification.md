### 1.3 Problems With Current Navigation

## 2. Navigation Architecture

### 2.1 Decision: Collapsible Left Sidebar + Slim Top Bar

**Replace** the current 64px top nav bar with:
- A **collapsible left sidebar** (240px expanded, 64px collapsed)
- A **slim contextual top bar** (48px)

**Rationale:**
- Sidebar accommodates 15+ items without clutter
- Grouping by function maps to the product's core loop (Discover → Analyze → Specify → Deliver → Learn)
- Matches PM tools users already know (Linear, Notion, Figma, Productboard)
- Collapsed icon-only mode preserves horizontal space on smaller screens
- Top bar becomes contextual (page title + breadcrumbs + page-level actions)

### 2.2 Sidebar Structure

The sidebar groups navigation items by the product's five-phase loop plus a workspace section at the bottom.

### 2.3 Sidebar Behaviors

| Behavior | Detail |
|---|---|
| **Expanded width** | 240px |
| **Collapsed width** | 64px (icon-only, tooltips on hover) |
| **Collapse trigger** | `[◀]` button in header, also toggleable via top bar hamburger `[≡]` |
| **Persistence** | Collapse state persisted in `localStorage` |
| **Section labels** | 11px, uppercase, font-weight 600, color `#5A5C66`, letter-spacing `0.05em`. Hidden when collapsed |
| **Active item** | `background: rgba(79,140,255,0.08)`, `color: #4F8CFF`, left border `2px solid #4F8CFF`, font-weight 600 |
| **Hover item** | `background: rgba(255,255,255,0.04)`, transition `150ms ease` |
| **Icon size** | 20px, inherits text color |
| **Item height** | 36px, padding `0 16px` |
| **Item font** | 14px Inter, weight 500, color `#B0B2BA` |
| **🔒 Locked items** | Same styling but with a tiny lock icon (12px) after the text. Clicking opens the "Coming Soon" overlay for that feature. Tooltip shows version: "Coming in v0.8" |
| **User menu** | Click avatar → flyout menu: Profile, Sign Out. Shows user name (14px, `#F0F0F3`) and email (12px, `#8B8D97`) |
| **Keyboard** | `Ctrl+B` / `Cmd+B` toggles sidebar collapse |
| **Mobile (< 768px)** | Sidebar becomes a slide-out drawer over the page. Triggered by `[≡]` hamburger in top bar. Backdrop overlay closes it |

- Consistent visual weight across all icons
- Variable font support (weight, fill, size adjustable via CSS)
- No emoji rendering inconsistencies across OS/browser
