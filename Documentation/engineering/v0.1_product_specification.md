## 1. Product Overview

### What It Does

Spec10x v0.1 lets product managers upload raw customer interviews (transcripts, audio, video) and get AI-synthesized insights — themes, pain points, feature requests, sentiment — without any manual tagging, organizing, or pre-categorization.

### Core Principles

| Principle | What It Means |
|---|---|
| **Zero-config start** | Upload files → get value. No setup wizards, no project creation |
| **AI organizes, not the user** | No projects, no folders. The AI discovers themes across ALL data |
| **Trust through transparency** | Every AI insight links to exact source quotes. Users can verify and correct |
| **Progressive disclosure** | Top-level dashboard is simple; depth is one click away |
| **Speed** | Everything feels instant — streaming responses, optimistic updates |

### What the User Gets

1. **Automatic theme discovery** — AI reads all interviews and clusters them into themes ranked by urgency and frequency
2. **Evidence trails** — Every theme links to exact quotes from exact interviews
3. **Sentiment analysis** — Know if feedback is positive, negative, or mixed
4. **Natural language Q&A** — Ask questions about your interviews in plain English (or any language)
5. **Export** — Get insights as structured markdown

## 2. User Journeys

### 2.1 New User — First-Time Flow

```
Sign Up (email/Google) 
    → Land on Insight Dashboard (empty state)
    → Click "Upload Interviews" 
    → Upload Modal opens
    → Drag & drop files
    → AI processes (transcription + analysis)
    → Metadata review step (optional, skippable)
    → Close modal → Dashboard populates with AI-discovered themes
    → Explore themes, read quotes, drill into interviews
    → Try "Ask Your Interviews" to query the data
    → Export insights as markdown/PDF
```

**Key moments:**
- **First wow:** Seeing themes auto-appear in the dashboard after uploading raw files
- **Trust moment:** Clicking a theme and seeing the exact quotes that support it
- **Power moment:** Asking a question in the Ask view and getting a cited answer

### 2.2 Returning User — Ongoing Flow

```
Log In 
    → Land on Insight Dashboard (populated)
    → See "New" badges on themes that emerged since last visit
    → Review new/updated themes
    → Upload more interviews (+ button in sidebar)
    → Dashboard auto-updates — themes evolve, new ones appear
    → Use Ask view for specific queries
    → Export updated insights
```

### 2.3 Upload Flow (detailed)

```
Click "+ Upload" (sidebar or empty state)
    → Upload Modal opens (large, overlay on dashboard)
    → Drag & drop files into drop zone (or click to browse)
    → Files appear in processing queue:
        Queued → Transcribing (audio/video) → Analyzing → Done
    → Live insight preview shows themes streaming in as files complete
    → All files processed → Metadata Review step appears:
        Each file listed by name with its detected/missing metadata:
        "interview_sarah.mp3 — We detected 2 speakers"
        "feedback_notes.pdf — No speakers detected"
        [Speaker 1: ___] [Role: ___] [Company: ___]
        [Skip — it's okay to leave these blank]
    → User fills in or skips
    → "Done — View Insights" button → Close modal → Dashboard updated
```

### 2.4 Exploration Flow

```
Dashboard shows top 10 themes (sorted by urgency × frequency)
    → Click a theme card → Right panel shows full detail:
        All quotes, sentiment breakdown, affected segments, related themes
    → Click "View full interview →" on a quote → Navigate to Interview Detail View
    → Read full transcript with color-coded AI highlights
    → Correct/adjust any AI extractions that seem wrong
    → Navigate back to Dashboard (breadcrumb)
    → Click "Show more themes" to see beyond top 10
    → Rename a theme via right-click or edit icon
```

### 2.5 Ask Flow

```
Click "Ask ✨" in top nav → Ask Your Interviews page opens
    → First-time: see suggested starter questions
    → Type a question: "What are the biggest pain points with onboarding?"
    → AI responds with structured answer + inline citation badges
    → Click a citation badge → jumps to that interview
    → See suggested follow-up questions below the response
    → Continue conversation with follow-up queries
```

## 6. Edge Cases & Rules

| Scenario | Behavior |
|---|---|
| **Duplicate file uploaded** | SHA-256 hash check → warn: "This file appears to already exist. Upload anyway?" |
| **Non-interview content** | Accept and process. If few/no insights found: flag as "Low insight content" in the sidebar |
| **Video/audio exceeds duration limit** | Block processing. Show plan limit + suggest uploading transcript instead |
| **Video processing** | Extract audio → speech-to-text → discard video. Only store transcript |
| **Contradictory feedback** | Show sentiment split on theme card. Highlight as "Polarizing theme" if split is close to 50/50 |
| **Sarcasm/misinterpretation** | AI flags low-confidence extractions with 🚩. User can manually correct |
| **Context collapse** | Quote detail always shows surrounding lines for context via "Show context" expand |
| **Theme with only 1 source** | Labeled as "Signal", hidden by default, visible with "Show all" |
| **Theme with 2+ sources** | Full "Theme" status, shown as a card |
| **Old themes (no new mentions)** | Preserved in "Previous themes" collapsible section with muted styling |
| **New themes (since last visit)** | "NEW" badge on theme card, auto-removed after user clicks the theme |
| **Delete interview** | Confirmation: "Deleting this will remove X insights and may affect Y themes." Recalculates themes |
| **Multilingual interviews** | Process natively. Present insights in user's interface language. Quotes in original language + "Translate" button |
| **Multi-speaker audio/video** | Speaker diarization → Speaker 1, 2, 3… User assigns names in metadata review |
| **Free tier limit reached (10 interviews)** | Upload button shows upgrade prompt. Cannot upload more until upgrading or deleting existing |
| **Theme rename** | Inline edit on theme card. Change reflected everywhere that theme is referenced |
| **Sample data** | "Try with sample data" loads a pre-built demo dataset so new users can explore before uploading |
