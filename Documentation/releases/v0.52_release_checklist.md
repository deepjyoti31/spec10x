## 1. Release Summary

**Release:** v0.52 — Analytics Source + Score v2
**Goal:** Add the first analytics source and improve score trust without over-claiming correlation.

Delivered:

- `signal_kind=metric_window` normalized shape for analytics data (`EPIC-052-01`)
- PostHog connector: validate, bounded weekly backfill (default 12 weeks, cap 26), normalize, hourly incremental sync (`EPIC-052-02`)
- Score v2: 14-day trend calculation, score-change explanations, source filter chips, correlational-language guardrails (`EPIC-052-03`)
- Hardening: bounded-cost design (one aggregate query per sync, event caps), replay fixtures, 25 new tests, full regression suite green (`EPIC-052-04`)

## 3. Rollout Notes

- `v0.52` rolls out to the same pilot accounts already receiving `v0.51` (Fireflies), since PostHog is additive to the existing feed/board/score surfaces rather than a separate workflow.
- No feature-flag gating beyond the existing per-connector connect flow — analytics connects the same way as any other source in the integrations shell.
- Rollback mechanism is identical to `v0.5`/`v0.51`: disable the connector for affected accounts, pause active `sync_runs`, roll back deployment if a defect is found post-release.

## 4. Post-Release Measurement Plan

Per `v0.5_project_tracker.md` §17, the `v0.52` measurement plan runs concurrently with the `v0.51` measurement window during pilot rollout (a deliberate decision, since both releases opened ahead of their formal gates):

| Metric | Target | Source Of Truth |
|---|---|---|
| Analytics sync success rate | ≥ 98% | `sync_runs` |
| Number of themes with both qualitative and quantitative evidence | Materially > 0 | analytics query |
| User comprehension of trend direction and score changes | Positive in pilot interviews | pilot interviews/survey |
| Feedback caused by confusing or misleading correlations | None / low | support log, pilot review |

If metrics come back weak, per the tracker's guidance: harden connector reliability before adding more analytics sources, and simplify score/trend UX before expanding further.
